  <action>
Create the health liveness endpoint per TechArch §2.1.2 and §4.3.

The handler lives at `app/api/health/route.ts` which maps to the URL path `GET /health` in Next.js App Router (note: `app/api/health/` → `/api/health` ... WAIT — re-check the spec):

Per TechArch §4.3 and §4.4, the path is `GET /health` (not `/api/health`). The file is listed as `app/api/health/route.ts` in §6.1, but the endpoint summary in §4.4 shows path `/health`. To produce the correct URL path `/health`, the file must be at `app/health/route.ts` (App Router maps `app/X/route.ts` → `/X`).

Create `app/health/route.ts` (NOT `app/api/health/route.ts`) so the URL is `/health`:

```typescript
import { NextResponse } from 'next/server';

export async function GET(): Promise<NextResponse> {
  return NextResponse.json({ status: 'ok' });
}
```

Per TechArch §4.3:
- Method: GET
- Path: `/health`
- Auth: None
- DB dependency: **None** — must NEVER open a database connection
- SLA: < 100 ms
- Response — 200 OK: `{ "status": "ok" }`

The handler must:
1. Return HTTP 200 status (NextResponse.json defaults to 200)
2. Return Content-Type: application/json
3. Return body `{ "status": "ok" }`
4. Have zero database dependency
5. Respond within 100ms (trivially met since there's no I/O)

Also create the directory `app/health/` if it doesn't exist.

NOTE: If the TechArch file path `app/api/health/route.ts` maps to `/api/health` in the App Router, prefer placing the file at `app/health/route.ts` to match the specified URL `/health`. The URL path is the authoritative spec; the file path is just an artifact. Confirm: in Next.js App Router, `app/health/route.ts` → URL `/health`. This is correct.
  </action>
